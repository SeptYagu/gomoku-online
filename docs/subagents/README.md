# 子代理报告目录

本目录用于归档研究、实现、验证兼记录、返工等子代理报告。所有子代理报告必须写成独立文档，不能只留在聊天记录里。

## 命名规范

推荐文件名：

```text
YYYYMMDD-HHMMSS-任务ID-代理角色-模块-agentId或昵称-状态.md
```

如果任务 ID 已经包含唯一日期编号，也可以使用短格式：

```text
YYYYMMDD-任务ID-代理角色-模块-agentId或昵称.md
```

## 文档开头

每份报告开头必须包含：

```text
时间编号：
任务 ID：
代理角色：
agent id：
昵称：
状态：完成 / 部分完成 / 失败 / 阻塞
写入范围：
```

代理角色：

- `研究`
- `实现`
- `验证兼记录`
- `返工`

其中，验证兼记录报告可以同时记录验证命令、浏览器检查、失败证据、验证矩阵更新、阶段报告验证状态更新和 `docs/HANDOFF.md` 报告路径更新。验证兼记录子代理不能修代码，不能修改实现内容；遇到 blocker 或 major 时，只写失败报告和返工建议，由主控派返工子代理处理。

## 最低内容要求

- 任务目标。
- 实际完成。
- 修改文件或阅读文件。
- 验证命令和结果。
- 失败证据或未覆盖项。
- 交给主控或下一位子代理的建议。

## 交接要求

主控更新 `docs/HANDOFF.md` 时，必须在文件末尾追加本轮记录，不能改写、删除、重排过去窗口的交接内容。追加记录必须包含本轮所有子代理报告文档路径。验证失败后的返工报告必须引用原失败报告路径；验证兼记录复测报告必须同时引用失败报告和返工报告路径。
