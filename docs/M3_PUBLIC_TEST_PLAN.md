# M3 Public Test Plan

更新日期：2026-06-25

M3 的目标是把好友房 MVP 放到真实服务器和真实设备上试跑，确认公开测试前不会被部署、联机、移动端误触或断线恢复问题卡住。完成 M3 后再进入 build plan 阶段 3：随机匹配、账号/游客身份、排行榜和离开惩罚。

## 依据

- `WEBSITE_BUILD_PLAN.md` 的 M3 定义：部署测试站，邀请用户试玩，记录移动端误触和断线问题。
- `docs/logic/realtime-room-module.md`：当前好友房为服务端权威状态机，Socket.IO room 只负责投递。
- `docs/logic/lobby-matchmaking-module.md`：同类项目的大厅、房间列表、找局、评分和离开惩罚只作为阶段 3 参考；M3 不提前引入账号或排行榜复杂度。

## 测试站

```text
http://gomoku.yagu.ddns-ip.net
```

每次部署后必须先确认实装版本：

```bash
npm run verify:online -- http://gomoku.yagu.ddns-ip.net <expected-version>
```

通过标准：

- `/en` 页面可访问。
- `/api/version` 返回 `<expected-version>`。
- Socket.IO polling 返回 sid。
- 强制 WebSocket transport 能连接成功。

## M3 完成门槛

M3 可以标记完成时，至少满足以下条件：

- 最新 `origin/main` 已部署到真实服务器，`verify:online` 全部通过。
- 两台不同电脑可以创建和加入同一好友房。
- 双方 ready 后自动开局，不需要 Start。
- 落子、回合切换、认输、终局重开在两端同步。
- 联机悔棋请求会出现在对手棋盘中央；允许、拒绝、10 秒自动拒绝都符合规则。
- 每人每局 3 次悔棋请求限制生效；同一局面被拒后不能连续请求。
- 刷新页面后，同 `playerId` 可恢复原座位。
- 对局中一方断线超过 60 秒后，在线对手自动获胜。
- 手机竖屏可以完成创建、加入、ready、落子、悔棋响应和重开；无明显文字溢出或按钮遮挡棋盘。
- 发现的问题已经记录到 `docs/M3_PUBLIC_TEST_LOG.md`，并按阻塞等级决定是否修复后重测。

## 测试流程

### 1. 部署验证

1. 用户部署最新 `origin/main`。
2. 记录目标短提交号。
3. 运行：

```bash
npm run verify:online -- http://gomoku.yagu.ddns-ip.net <expected-version>
```

4. 若 version 失败，先确认服务器是否已拉取最新代码并重新 `npm run build && npm start`。
5. 若 polling 或 websocket 失败，优先查 Nginx/OpenResty `/socket.io/` 代理和 upgrade header。

### 2. 双电脑好友房

1. D1 打开 `/en`，进入 Friend room。
2. D1 创建房间，记录 room code。
3. D2 使用邀请链接或 room code 加入。
4. D1、D2 都点击 Ready。
5. 确认无需 Start，棋局直接开始。
6. D1 落中心点，D2 确认同一坐标出现黑棋。
7. D2 落一手，D1 确认同步。
8. 当前回合之外的一方尝试点击空点，应不能落子。

### 3. 联机悔棋

1. 当前最后一手落子者点击 Undo。
2. 对手棋盘中央出现请求弹框。
3. 点 Allow，确认棋盘回退、回合正确。
4. 再发起请求，点 Reject，确认棋盘不变。
5. 同一局面再次请求，应被拒绝为同局面连续请求。
6. 再发起可用请求后不操作，等待 10 秒自动拒绝。
7. 同一玩家每局第 4 次请求应失败。

### 4. 刷新和断线

1. 对局中 D1 刷新页面。
2. D1 应恢复同一房间、同一座位和已有棋子。
3. 对局中关闭 D2 浏览器或断开网络。
4. D1 应看到 D2 disconnected。
5. 等待超过 60 秒。
6. D1 应自动收到胜负结果。

### 5. 移动端误触

1. M1 打开 `/en`。
2. 创建或加入好友房。
3. 检查功能栏、棋盘、玩家列表和弹框是否互相遮挡。
4. 在棋盘中心、边线、角落各落子一次。
5. 检查点击是否落在预期交点。
6. 触发悔棋弹框，确认 Allow/Reject 倒计时按钮可点且不挡住关键内容。

## 阻塞等级

| 等级 | 定义 | 处理 |
| --- | --- | --- |
| P0 | 无法访问网站、无法创建/加入房间、构建失败、棋局状态严重错乱 | 立即停止推进阶段 3，先修复 |
| P1 | 联机核心路径失败：ready、落子、悔棋、刷新恢复、断线判负任一不可用 | 修复后重跑对应流程 |
| P2 | 移动端布局、文案、轻微误触、非核心按钮状态问题 | 记录并尽量在 M3 收口前修 |
| P3 | 文案润色、视觉细节、低频体验问题 | 可进入后续阶段排期 |

## 记录规则

- 所有公测问题写入 `docs/M3_PUBLIC_TEST_LOG.md`。
- 每条问题必须包含版本号、设备、浏览器、网络、复现步骤、期望、实际、阻塞等级和状态。
- 修复后在同一条问题下追加验证记录，不删除原始记录。
- Handoff 只追加新窗口记录，不修改过去窗口内容。
